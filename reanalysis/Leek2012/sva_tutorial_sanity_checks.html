Sanity check for ComBat in the SVA tutorial.

2014-11-21 15:21:15

Overview

This small report aim to show that the application of ComBat as described in the user guide accompanying the R package sva presented in Leek et al. will lead to false results even for random data.

One sanity check will be performed

  • Real data will be replaced with random numbers and a batch effect will be added wherafter the rest of the tutorial will be performed with a special focus on the procuced p-values.

Most code are taken directly from the vignette source ‘vignettes/sva/inst/doc/sva.Rnw’. For more explanation, please read the sva tutorial in parallel while remembering that the real data is replaced with random numbers.

This report is based on version 3.8.0 of sva. Version 3.12.0 does no longer include covariates when illustrating ComBat use..

In this sanity check the real data is replaced with random numbers. A batch effect is also added since sva needs a batch effect or it will report “No significant surrogate variables”. This is a simple and not necessarily the most realistic way of adding a batch effect.

set.seed(100)
edata[,] = rnorm(length(edata))
for(batch in (unique(pheno$batch)))
{
    for(s in 1:dim(edata)[1])
    {
        thisgenesbatcheffect = rnorm(1)
        a= pheno$batch==batch
        edata[s, a] = edata[s, a] + thisgenesbatcheffect
    }
}

Next, the ComBat method is applied, where the batch effect is estimated while trying to retain the group differences. The data is adjusted and in the next step a differential expression analysis is performed.

batch = pheno$batch

mod = model.matrix(~as.factor(cancer), data=pheno)

combat_edata = ComBat(dat=edata, batch=batch, mod=mod, numCovs=NULL, par.prior=TRUE, prior.plots=FALSE)
## Found 5 batches
## Found 2  categorical covariate(s)
## Standardizing Data across genes
## Fitting L/S model and finding priors
## Finding parametric adjustments
## Adjusting the Data
pValuesComBat = f.pvalue(combat_edata,mod,mod0)
qValuesComBat = p.adjust(pValuesComBat,method="BH")
hist(pValuesComBat,breaks=100)

The p-values from the analysis when ComBat adjusted data is used shows a rather huge skewness, and many genes would falsely have been reported as differentially expressed.

The next example is in the tutroial described as a simplified version of ComBat.

modBatch = model.matrix(~as.factor(cancer) + as.factor(batch),data=pheno)
mod0Batch = model.matrix(~as.factor(batch),data=pheno)
pValuesBatch = f.pvalue(edata,modBatch,mod0Batch)
qValuesBatch = p.adjust(pValuesBatch,method="BH")
hist(pValuesBatch,breaks=100)

The above method of handling batch is quite similar to the blocking method described in the limma guide. The resulting p-values are as expected for random data.
